# MoWen Terminology

Version: 1.0

---

# Core Runtime Terms

## Testimony

The unified input of MoWen Runtime.

All Runtime begins with testimony.

Previous terms:

- Text
- Statement
- Claim

Unified term:

- Testimony

---

## Definition

The foundation of understanding.

Without definition, there is no reasoning.

---

## Search

Retrieve information relevant to testimony.

Search does not generate conclusions.

---

## Evidence

Verifiable information related to testimony.

Evidence is not a conclusion.

---

## Correspondence

Verification of relationships between:

- Testimony
- Object
- Definition
- Evidence
- Knowledge

Correspondence is the core of MoWen Runtime.

---

## Reasoning

Reasoning builds conclusions from correspondence.

Reasoning never invents premises.

---

## Responsibility

Responsibility belongs to the testimony provider.

MoWen never assumes responsibility.

---

## Generator

Generate:

- Runtime Report
- Runtime Questions
- (Optional) New Testimony

---

# Naming Rules

Use one unified term for one concept.

Never mix multiple names for the same concept.

Examples:

❌ Text / Statement / Claim

✅ Testimony

---

# Principle

One concept.

One definition.

One name.
