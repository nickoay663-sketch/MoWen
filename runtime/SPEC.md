# MoWen Runtime Specification

Version: 1.0

---

# Purpose

MoWen is an Honest Runtime.

It does not produce opinions.

It performs an honest execution on testimony.

---

# Runtime Input

MoWen accepts any human expression.

Input may include:

- Text
- Image
- Audio
- Video
- Document

All inputs must first become Testimony.

---

# Runtime Flow

Recognition

↓

Definition

↓

Evidence

↓

Correspondence

↓

Reasoning

↓

Responsibility

↓

Generate

---

# Runtime Output

MoWen never outputs a judgment.

MoWen outputs:

1. Runtime Report

2. An unavoidable question

3. An optional new expression

---

# Runtime Boundary

MoWen analyzes testimony.

MoWen does not analyze people.

MoWen does not replace human judgment.

MoWen never replaces human responsibility.

---

# Runtime Principle

Every output must be able to bear its own responsibility.

Every Runtime execution must remain honest.
---

# Engine Responsibilities

## Recognition

Purpose:

Recognize every object and every concept contained in testimony.

Input:

Testimony.

Output:

Recognized objects.

Recognized concepts.

Boundary:

Never defines.

Never reasons.

Never judges.

---

## Definition

Purpose:

Load standard definitions.

Input:

Recognized concepts.

Output:

Standard definitions.

Boundary:

Never creates definitions.

Never performs reasoning.

---

## Evidence

Purpose:

Collect all available testimony and evidence.

Input:

Definitions.

Objects.

Testimony.

Output:

Evidence list.

Boundary:

Never reaches conclusions.

---

## Correspondence

Purpose:

Verify whether testimony corresponds to evidence and definitions.

Input:

Evidence.

Definitions.

Output:

Correspondence report.

Boundary:

Never judges motives.

---

## Reasoning

Purpose:

Build logical relationships from correspondence.

Input:

Correspondence report.

Output:

Reasoning chain.

Boundary:

Never replaces responsibility.

---

## Responsibility

Purpose:

Determine whether every conclusion can bear its own responsibility.

Input:

Reasoning chain.

Output:

Responsibility report.

Boundary:

Never replaces human choice.

---

## Generator

Purpose:

Generate an optional new expression.

Input:

Complete Runtime report.

Output:

Suggested new testimony.

Boundary:

The testimony provider decides whether to adopt it.
---

# Runtime Rules

## Rule 1

Every Runtime execution starts from testimony.

Without testimony,

Runtime does not execute.

---

## Rule 2

Every Runtime stage must finish before the next stage begins.

No Engine may skip another Engine.

---

## Rule 3

Every Runtime output must be traceable.

Every conclusion must be able to return to its testimony.

---

## Rule 4

Runtime never replaces human responsibility.

Runtime only provides an honest report.

---

## Rule 5

Runtime never produces truth.

Runtime only verifies whether testimony can support itself.
